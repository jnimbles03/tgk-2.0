Architecture

Five files cooperate via URL params. Narration lives in one place; visual furniture is shared across all 23 verticals.

1 · Entry
↓
/picker.html  or direct link
User chooses vertical + flow. Picker is canonical; geos.html and index-unified.html redirect here.
2 · Per-vertical entry
↓
/stories/story-<vertical>/index.html
One-line meta-refresh redirect. Never edit narrative here.
3 · Shell
↓
/stories/_shared/story-shell.html ?vertical= &usecase= &splash=
Reads URL, looks up VERTICALS[key], renders sidebar copy + headlines + persona chips + beats. Iframes scene templates.
4 · Scene template
↓
/story-templates/docusign-<scene>.html ?preset= &embed=1
Visual UI moment — Agreement Desk · CLEAR · Signing · Agreement Manager · Workspace. Shared across all verticals.
5 · Preset lookup
↓
PRESETS map inside the scene template
Workspace template carries 21 presets (tenant brand, sections, agent rail). Other templates use preset for branding only.

Key insight: narration lives in one place — the VERTICALS map inside story-shell.html. Scene templates are visual furniture; they only know which vertical they're rendering when the shell hands them ?preset=.

Setup

Day-one: clone, install, run locally. ~10 minutes.

01

Clone the repo

Repo: jnimbles03/tgk-2.0 (private). Mac working copy by convention at ~/Documents/Claude/Projects/TGK 2.0.

git clone git@github.com:jnimbles03/tgk-2.0.git
cd tgk-2.0
Don't edit in Replit. Mac is the source. GitHub is truth. Replit deploys. Editing in the Repl creates drift you can't pull back.
02

Install and run

Node 18+. Express server. No build step.

npm install
npm start
# → server on http://localhost:3000

Confirm with these URLs:

  • http://localhost:3000/ → redirects to picker
  • http://localhost:3000/stories/_shared/story-shell.html?vertical=wealth
  • ...?vertical=wealth&splash=1 — splash on
  • ...?vertical=banking-deposits&usecase=maintenance — alt flow
Backend routes only run locally or on Replit. The feedback widget POSTs to /api/feedback — that's an Express route, not a Pages target. On the GitHub Pages mirror, those POSTs 404 silently. Don't chase that as a bug.

Make your first edit

The full path: open the file, change a word, refresh, see it.

03

Edit a tenant string

  • Open stories/_shared/story-shell.html
  • Search for wealth: { — should land near line 772
  • Find tenant: "Hillside" a few lines down
  • Change it to "Hillside Advisors"

Deploy

Two targets, one source. Don't fight it.

04

Understand the deploy path

  • Replit — full Express server. Backend feedback API works. Auto-pulls from GitHub main.
  • GitHub Pages (jnimbles03.github.io/tgk-2.0/) — static-only mirror. Auto-deploys on push to main via .github/workflows/deploy-pages.yml.

The Pages workflow runs .github/scripts/rewrite-paths.mjs which prepends /tgk-2.0/ to every root-relative path — that's why source files use absolute paths like /assets/... instead of relative.

Never convert source paths to relative. Both targets depend on the absolute form. Pages rewrites; Replit serves as-is. Relative paths break Pages.
05

Commit and ship

Direct commits to main are the working pattern today. See Operations for the verify-before-push checklist.

git add -p
git commit -m "wealth: tenant string update"
git push origin main
# Replit auto-pulls within ~30s
# Pages workflow deploys static mirror within ~2min
Sharp edge: the youtube-demos/ folder is gitignored — ~7 GiB of reference videos. A 2026-04-25 push hit GitHub's HTTP limit when this rule was missing. Keep reference media gitignored.

01 · URL contract

Every story is fully determined by its URL. The shell parses these on load and reads nothing else for vertical selection.

ParamTypeEffect
verticalstring · requiredKey into VERTICALS. 23 valid keys today.
usecasestring · optionalOne of: maintenance, fraud-fabric, intake-3, intake-4, workspaces, sales-deepdive, procurement-deepdive. Omit for canonical onboarding. auth-fabric aliases fraud-fabric.
splash"0" | "1"Force Workspace splash on/off. Omit to use per-vertical splashOk default.
presetstring · injectedPassed to scene templates by the shell. Don't set manually.
embed"1" · injectedTells a scene template to hide its outer chrome.
modestring · per usecaseVariant inside one template. From USECASE_TEMPLATE_MODES.

02 · VERTICALS entry

The full shape of one vertical block. Where 95% of narrative edits land.

File:stories/_shared/story-shell.htmlLine:771
"banking-deposits": {
  label:        "Banking Deposits",
  tenant:       "Cedar Federal",
  subtitle:     "× Williams Household — …",
  tenantColor:  "#6BB6FF",
  preset:       "banking-deposits",        // passed to iframes as ?preset=
  category:     "onboarding",              // onboarding | maintenance — picker chip filter
  splashOk:     true,                       // optional · Scene 5 splash default

  scenes: [ exactly 5, see Scene schema ],

  usecases: { optional · alt narration per ?usecase= 
    maintenance:    { scenes: [ 3 scenes ] },
    "fraud-fabric": { scenes: [ 3 scenes ] },
    workspaces:     { scenes: [ 1 scene  ] }
  }
}

Required: label, tenant, subtitle, tenantColor, preset, category, scenes (exactly 5). Optional: splashOk, usecases. The preset value must match a key in workspace PRESETS.

03 · Scene

One element of scenes[]. Five per canonical onboarding (sender · identity · signing · data · workspace).

{
  tag:     "SCENE 1 OF 5 · INTAKE · SENDER WEBFORM",  // eyebrow
  head:    "Cathie Woods at Hillside is bringing TGK Capital onboard…",
  persona: { side:"advisor", name:"Cathie Woods", role:"Director · Hillside" },
  lede:    "<p>Cathie opens a new sub-advisor engagement…</p>",  // HTML
  beats:   [ exactly 5 Beat objects ],
  src:     "/flipbooks/…"                              // optional · only for stage-iframe scenes
}

04 · Beat

One of scenes[i].beats[]. Five per scene; each corresponds to a hotspot inside the iframed template.

{
  head:    "Patrick adopts his signature.",
  lede:    "<p>The Adopt Signature modal shows a typed preview…</p>",
  persona: { side:"system", name:"AI assistant", role:"AI · Grounded Answer" }
            // optional · overrides scene persona for this beat only
}

05 · Persona

Speaker chip. Same shape on a Scene or a Beat.

{
  side: "advisor",        // "advisor" | "client" | "system" | "handoff"
  name: "Cathie Woods",
  role: "Director of Advisor Onboarding · Hillside"
}

advisor = the firm's person · client = the customer · system = automation / AI · handoff = transition between actors.

11 · postMessage protocol

How the shell drives an iframed scene template or vignette. Implement these when authoring a new template.

// Shell → iframe
iframe.contentWindow.postMessage({ type: "tgk:lockToBeat", view: "appcenter", step: "done" }, "*");
iframe.contentWindow.postMessage({ type: "tgk:setPreset",  preset: "insurance-life" }, "*");
iframe.contentWindow.postMessage({ type: "tgk:queryRect", name: "cf_step1" }, "*");
iframe.contentWindow.postMessage({ type: "tgk:advanceTo", sceneId: "s3" }, "*");

// Iframe → shell (response to queryRect)
parent.postMessage({ type: "tgk:rect", name: "cf_step1", x: 0.42, y: 0.58, w: 0.18, h: 0.06 }, "*");

// Hotspot anchor on a sub-element inside the iframe
<button data-hotspot="cf_step1">…</button>

    Operations

    Verify, audit, deploy. The pre-push checklist that keeps a vertical from shipping broken.

    Smoke-test Audit Calibrate Feedback Pre-push Conventions
    01

    Smoke-test before every push

    Three URLs. Two minutes. Catches 90% of broken-JSON, missing-preset, and misnamed-key bugs.

    • Onboarding: ?vertical=<key> — click through all 5 scenes × 5 beats. Sidebar and iframe should advance together.
    • Workspaces: add &usecase=workspaces — Scene 5 only, populated from PRESETS.
    • Splash: add &splash=1 — ~10s splash before Scene 5.
    Symptom: sidebar reads as the wrong vertical's content. Cause: JSON shape error in VERTICALS.<key>. Shell silently falls back to 'wealth' on lookup miss.
    02

    Run the persona audit

    Mechanical pass: fonts, colors, emoji, structure, persona consistency across the cast.

    bash _audits/_persona_audit_build.sh
# → regenerates audit pages under _audits/
# → open _audits/index.html locally to review

    Last green run: 74/74 OK on 2026-04-30. Common failures: emoji in narration (use SVG); persona side not in enum; tenant color hex changed without updating chrome.

    03

    Hotspot calibration

    If a tooltip is pinned to the wrong spot inside the iframe, use calibrate mode instead of editing coordinates.

    • Append &calibrate=1 to any shell URL.
    • Each hotspot becomes draggable. Drag to anchor.
    • New coords store in localStorage tgkAdmin.hotspots.
    • Export the override and paste into the shell's hotspot map.

    Hotspots are scene-keyed, not vertical-keyed. Recalibrating once fixes it everywhere.

    04

    Feedback pipeline

    Feedback widget on every page; submissions land in Replit DB; admin reviews at /admin/feedback.

    • POST /api/feedback — public. AI triage runs in background.
    • /admin/feedback — Basic Auth (admin + ADMIN_PASSWORD env). Live updates via SSE.
    • /api/feedback/public-<slug> — CORS-open, PII-stripped. Used by Cowork artifact.

    Auto-fix execution is intentionally not wired. Approvals are recorded; execution stays manual.

    05

    Pre-push checklist

    • Smoke-test the affected vertical (3 URLs above).
    • Touched persona, casting, or cross-vertical patterns? Run _persona_audit_build.sh.
    • Added a new vertical? Confirm three coordinated edits — VERTICALS, PRESETS, redirect file.
    • Added or modified a scene template? Spot-check at least three verticals that use it.
    • Diff for accidentally-committed reference media or screenshots over a few MB.
    • Push. Replit pulls in ~30s; Pages deploys in ~2min.

    Conventions

    Things easy to forget when you're deep in a file.

    • No emoji. Use inline SVG or brand icons. The persona audit catches strays.
    • TGK Capital is a character, not a co-brand. Appears inside wealth + banking storylines as the customer firm. Patrick Meyer = TGK signer. Never appears as site chrome.
    • SoR badge per vertical: Guidewire PAS for FINS insurance, nCino for banking-deposits.
    • Source paths are root-relative (/assets/...). Pages workflow rewrites; Replit serves as-is. Never convert to relative.
    • Front-office role vocabulary is per-domain. Insurance: Agent = Advisor. Compliance/legal/ops are never advisor-side.
