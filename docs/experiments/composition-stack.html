TGK 2.0 / Architecture

From 198 component mocks up to one picker.

A six-layer stack. Each tier composes the one below it. The story shell is the keystone: a single 10,185-line file that, given ?vertical=&usecase=, selects which scene templates load as iframes — replacing what would otherwise be a hundred hand-built pages.

Tier 1 — Components
The atoms. Each file mocks a single vendor system tile (CRM, doc assembly, EHR, claims admin) at current-state or future-state.
Tier 2 — Vendor portals
Replicas of real vendor UIs (Emptoris today; more via the tgk-vendor-portal skill) that host Docusign templates as drill-downs.
Tier 3 — Story templates
The 12 reusable Docusign scene moments. Agreement Desk is the Workflow Builder orchestration visualizer; Workspace can layer the Mastercard splash.
Tier 4 — Story shell
One file. Reads ?vertical= for tenant brand and ?usecase= to swap the iframe sequence — 18 usecase variants in flight.
Tier 5 — Redirect stubs
23 one-line files that exist only to set the ?vertical= param and forward to the shell.
Tier 6 — Picker
The storefront. ~100 entry rows across 20 verticals × ~5 flows, plus Headless IAM shortcuts.
