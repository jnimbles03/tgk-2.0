Stage 1: Decode

Input: MP4 file

Tool: ffmpeg

Output: frame PNG sequence (user-specified fps)

Gating: Max 600s, max 2GB

Stage 2: Classify

Input: Frame sequence

Decision: REPLAY (≥8s kept), SYNTHESIZE (<8s), or HYBRID

Output: segments.json, timeline.json (8 primitives)

Model: Gemini 2.5 (stubbed)

Stage 3: Extract

Input: Segments, timeline

Output: structure.json, content.json, theme.json

Key rule: No vendor-specific data in structure — forces verticalization reusability

Stage 4: Render

Input: Structure, content, theme

Output: Self-contained build/index.html (≤5MB)

Optional: Inline or file-ref assets

Scene 1 Archetype Framework

Scene 1 intake takes one of four shapes per vertical. The rest of the 5-scene spine holds constant.

A: Sender Webform

Employee at tenant fills an internal webform. Maestro packages and routes. Example: wealth (Cathie at Hillside onboards TGK). Tag: SENDER WEBFORM

A-Trig: Maestro-Triggered

System event (renewal date, cert expiry, check clear). Maestro auto-triggers workflow. Example: slgov-recertification (annual Medicaid). Tag: MAESTRO-TRIGGERED

B: Self-Service

Customer/applicant fills public/embedded webform. Maestro routes back to sender. Example: insurance-pc (claimant reports loss). Tag: SELF-SERVICE

C: Inbound

Genuine inbound submission (death notice, external request). Agreement Desk visualizes mailroom. Example: insurance-life (beneficiary claim). Tag: INBOUND

D: Workspace-Led

Non-linear engagement (commercial loan, family wealth). Workspace deal room opens first; webforms fire inside. Example: banking (Atlas loan). Tag: DEAL ROOM

3. Decision Log

Key architectural and content decisions that define the system. All are in active use as of 2026-05-07.

60-cell canonical grid (4×3×5 locked)

Every sub-vertical exposes exactly 5 demos (Onboarding, Maintenance, Fraud Fabric, Headless IAM, Workspaces). 4 clusters × 3 subs = 60 cells. The 11 split-out vertical keys live as cell content, not top-level picker entries. URLs as deep-links still work.

HTML-first, no runtime JSON or DB

All demo data (VERTICALS, PRESETS, narration, hotspots) live as inline JavaScript objects in HTML files. Python runs at build-time only (frame extraction, flipbook scaffolding). No config server, no runtime DB queries.

Variables injected per preset (ad-lib model)

Tenant name, customer name, color, tenantLogo, customerLogo, SoR badge, signature moment all injected via ?preset= parameter. Same shell renders any vertical by swapping variables. Enables rapid re-skinning.

Hotspots scene-keyed, not vertical-keyed

Tooltip coordinates are shared across all verticals because they target Docusign template UI that doesn't move per preset. Only narration varies by vertical. Reduces coordinate maintenance burden from 23×5 to 5 unique hotspot maps.

Shell is the only place narration lives

VERTICALS map in story-shell.html is the single source of truth for all beat headlines, ledes, persona chips. Scene templates are visual furniture; they don't author narrative. Edits flow through story-shell only — no scattered narrative fragments.

4 + 1 Scene 1 archetypes (A, A-trig, B, C, D)

Scene 1 entry takes one of five shapes: sender webform, Maestro-triggered, self-service webform, inbound Agreement Desk, or Workspace-led. Scenes 2–5 hold constant. Allows one shell to express vastly different intake patterns across domains.

Agreement Desk = Maestro proxy, not literal actor

Agreement Desk visualizes invisible Maestro orchestration. When authoring narration, frame Maestro as the actor ("Maestro receives and routes…") and Agreement Desk as the visualizer ("…made visible on screen here."). This keeps the same template valid across intake archetypes.

Workspace splash via ?splash=1 (MasterCard-style intro)

Optional 360ms violet sweep + headline on Scene 5 entry. MasterCard-style voiceover pause (now indefinite, press Space to dismiss). Can be enabled per-vertical via splashOk: true or forced via URL param.

SoR badge sweep: Guidewire (FINS insurance) + nCino (banking-deposits)

Persistent "System of record: {vendor}" pill in scene chrome. Reads as "Docusign wraps around these core platforms." Scope: FINS insurance + banking-deposits only. Other domains out of scope.

TGK Capital is a character, not a co-brand

TGK Capital appears inside wealth + banking demo narrations as the customer firm being onboarded. No co-brand chrome anywhere. Patrick Meyer is the TGK signer. Topology: Docusign chrome only; TGK lives in the fiction.

No emoji anywhere. Inline SVG or brand icons only.

TGK 2.0 maintains a Docusign-themed premium aesthetic. Emoji undermine brand polish. All UI icons are inline SVG (feather-style, 1.5px stroke) or Docusign iconography.

Cowork-on-Mac is the only edit surface

Source of truth lives at /Users/patrickmeyer/Documents/Claude/Projects/TGK 2.0 on Patrick's Mac. GitHub is the commit source. Replit deploys. Never edit in the Repl — changes are lost on restart.

Cowork sandbox can't push to GitHub. Manual handoff required.

Commits happen in Cowork, but the sandbox has no GitHub credentials. Final git push happens from Mac terminal. Watch for orphaned .git/*.lock files after failed pushes.

Admin auth via single password (Welcome01!) + cookie

All /admin/* routes + builder Stage 4 + audit-grid edits gated by /assets/admin-auth/ primitive. Prompt stores x-tgk-admin cookie in sessionStorage. Single key currently; role-based gating deferred.

5-scene spine: Agreement Desk → CLEAR → Signing → Navigator → Workspace

Canonical ordering across all onboarding demos. Only Scene 1 archetype varies. Other flows (maintenance, fraud, intake, workspaces) pick different scene-keys arrays but when all 5 scenes are needed, this is the order.

Persona handoff via autoplay-only with 360ms violet sweep

At persona transitions, shell fires a violet flash + headline emphasis. No click-to-handoff pause. Autoplay continues. Same-persona beats stay flat (no sweep).

Speed baseline retuned: SPEED_MULTIPLIER = 1.75

Effective playback rates are 1.75× / 2.625× / 3.5× (vs. displayed 1× / 1.5× / 2×). Cold-open default dropped from 1.5 to 1.0 since baseline is now faster.
